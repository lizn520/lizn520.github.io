EmguCV 求图像重心，很简单的函数

发表于 2015-09-18 | 更新于 2018-04-22 | 分类于 图像处理 ， 扯淡 | 评论数：

在网上找了几天资料看了些例程，总算理解了用“矩”来求重心的算法。

大多数资料的表述如此：

“不规则区域的矩，表示把一个归一化的灰度级图像函数理解为一个二维随机变量的概率密度。”（我感觉这句话怪怪的，吓得我回去好好翻了翻概率论书，我觉得应该这样改成“把一个归一化的灰度级图像函数理解为一个二维随机变量的概率密度。”然后我高兴的觉得自己理解这东西了~）

这个随机变量的属性可以用统计特征–矩（Moments）来描述。通过假设非零的像素值表示区域，矩可以用于二值或灰度级的区域描述。

Mpq = sigma(i)sigma(j) ip jq f(i,j)

其中x,y,i,j是区域点的坐标（在数字图像中的像素坐标）。

令Xc,Yc表示区域重心的坐标，则：

Xc = M10/M00;

Yc = M01/M00;

在二值图像的情况下，M00表示区域的面积。

对了，我用的是emgucv3.0的库。

 

在emgucv中最关键的就是这两句：

MCvMoments moment;
moment = CvInvoke.Moments(tem.Convert<Gray,Byte>(). ThresholdBinary(th2,maxth2));

这是用emgucv库函数计算各阶矩的函数，把二值化图像作为参数传入，返回

MCvMoments类型

然后重心点就可以算出来了

MCvPoint2D64f point2d64f;

point2d64f=moment.GravityCenter;
亦可以这样算
